File 4: UI/UX Design Brief
Project Name: Janus
Art Style Direction: Retro-Systems Utility / Windows 95 Toolkit (Y2K CRT HUD)
1. Style Overview & Core Philosophy
The visual layout for Janus must explicitly reject modern, minimalist web aesthetics (no oversized whitespace, no soft gradients, no blurry rounded card corners). Instead, it must capture the high-density, hyper-functional aesthetic of a late-90s or early-2000s desktop operating system utility—specifically channeling a dark-themed Windows 95/98 or classic CRT network diagnostic suite.

The interface should feel close-to-the-metal, tactical, and dense with information. It is a tool built by systems engineers, for systems engineers. It should look like an authoritative administrative dashboard pulled straight out of a vintage sci-fi or retro-computing control room.

2. Visual Inspiration & Mood Library
The AI agent should loosely reference the following design cues, leveraging its activated @frontend and @ui-patterns skill modules:

Classic Operating Systems: Gray/slate beveled window borders, prominent blue block-colored title bars, and retro window navigation buttons (Minimize _, Maximize ▢, Close ✕).

The Classic Start Bar: A permanent system utility bar at the top or bottom of the screen featuring a classic "Start" style launcher button, system clock readouts, and operational state chips.

CRT Instrument Panels: Monospace grid matrices, digital terminal feeds, and glowing phosphor graph lines that mimic real-time hardware telemetry.

3. Visual Identity, Palette & Typography (Loosely Enforced)
The agent is free to dynamically balance the contrast, but should stick close to this retro industrial color space:

Dominant Surface Colors: Deep obsidian black or solid charcoal (#121212) for master panel canvas layers, with window frames styled in dark slate blues or classic retro grays.

Illumination Accents: * Phosphor Mint Green: For active traffic streams, healthy metrics, and ticking connection clocks.

Neon Pink / Laser Magenta: For fault-injected elements, broken sockets, and critical HTTP 429 rate-limit alerts.

Command Blue: For primary frame headers and focus indicators.

Typography: Strict system monospace families across the entire viewport (e.g., 'JetBrains Mono', 'Fira Code', 'Courier New', or fixed-width fallbacks). Use capitalized headers with slight letter-spacing to reinforce the tactical instrument cluster aesthetic.

4. Loose Structural Blueprint
The layout should occupy a single viewport without vertical scrolling, distributing metrics into clear, blocky utility windows. The agent is encouraged to organize these components fluidly based on available space:

The Packet Ledger (Left): A high-density, scrollable terminal frame outputting live network connection records, raw client IPs, timestamp strings, and token bucket balances.

The Topography Console (Center): A raw node chart mapping network flow. Janus sits as a retro desktop computer console or server tower, distributing data paths to the target server ports, visibly shifting into a highlighted neon pink/magenta alert state when a simulated error is toggled.

The Analytics Cluster (Right): Sharp, rigid, grid-backed line and bar graphs tracking backend server metrics (Average Latency, Packet Loss, and Upstream TCP Connection Pool Utilization percentages).

5. Execution Directives for the AI Agent
Instruction to the Agent: Invoke your freshly indexed @frontend, @theme-factory, and @ui-skills playbook bundles to implement custom CSS utility structures. Do not fall back on soft, generic consumer app styles. Create crisp pixel borders, flat drop shadows, and text elements that emulate old-school terminal screens. Prioritize technical information density and rich status reporting. You have full creative autonomy to arrange widgets and map responsive breakpoints, provided the end result strictly honors this Y2K retro-systems utility identity.
